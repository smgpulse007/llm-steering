# Optional VHS tape workflow

This repository ships **verified portable GIF assets** in `docs/assets/` that are generated by `scripts/build_showcase.py`.

If you prefer true terminal-recorded demos, the open-source [VHS](https://github.com/charmbracelet/vhs) tool is a strong option for scripting terminal GIFs as code.

## Why this directory exists

VHS is excellent for terminal-native recordings, but shell behavior can vary across Windows environments. Rather than make the public README depend on a brittle local shell capture path, this repository treats VHS as an **optional enhancement** while keeping the checked-in GIFs reproducible on the verified local setup.

## Suggested local setup for VHS

On Windows, a typical path is:

- install `vhs`
- ensure `ffmpeg` is on `PATH`
- ensure `ttyd` is on `PATH`

Example installation path used during research:

- `scoop install vhs`

## Recommended use

Use VHS locally if you want to record:

- setup walkthroughs
- terminal-only demos
- alternative themes or window chrome
- recordings of custom prompt sweeps

For the public repo, the primary supported assets remain the generated GIFs in `docs/assets/` and the compact JSON files in `docs/showcase/`.
